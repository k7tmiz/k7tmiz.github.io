Git 常用命令详解

发表于 2025-06-07 分类于 Tech

Git 是一个强大的分布式版本控制系统，以下是核心命令的简明介绍：

1. git add - 添加文件到暂存区

  • git add <file>: 添加指定文件。
  • git add .: 添加所有新建、修改、删除的文件（不含忽略文件）。
  • git add -u: 添加所有已跟踪文件的修改和删除。
  • git add -A: 添加所有新建、修改、删除的文件（包括未跟踪文件）。
  • git add -p: 交互式选择文件中的修改块。

2. git commit - 提交到本地仓库

  • git commit -m "Your message": 提交暂存区的修改，并附带提交信息。
  • git commit: 打开编辑器输入详细提交信息。
  • git commit -a -m "Your message": 跳过 git add -u，直接提交所有已跟踪的修改。
  • git commit --amend: 修正上一次提交（慎用已推送的提交）。

3. git push - 推送到远程仓库

  • git push <remote> <branch>: 将本地分支推送到远程仓库。
  • git push -u <remote> <branch>: 首次推送时，设置本地分支与远程分支的跟踪关系。
  • git push: 如果已设置跟踪，直接推送。
  • git push --force: 强制推送（极度危险，会覆盖远程历史）。
  • git push --delete <remote> <branch>: 删除远程分支。

4. git remote - 管理远程仓库

  • git remote: 列出所有远程仓库。
  • git remote -v: 列出远程仓库及URL。
  • git remote add <name> <url>: 添加新的远程仓库。
  • git remote rm <name>: 删除远程仓库。
  • git remote set-url <name> <new_url>: 修改远程仓库URL。

5. git rm - 从仓库中删除文件

  • git rm <file>: 从工作目录和暂存区删除文件。
  • git rm --cached <file>: 从暂存区删除文件，但保留工作目录中的文件。
  • git rm -r <directory>: 递归删除目录。

6. git pull - 拉取并合并远程修改

  • git pull <remote> <branch>: 拉取并合并指定远程分支。
  • git pull: 如果已设置跟踪，直接拉取并合并。
  • git pull --rebase: 拉取后进行变基（推荐，保持线性历史）。
    • git fetch: 只拉取远程修改到本地，不自动合并。

7. git merge - 合并分支

  • git merge <branch_name>: 将指定分支合并到当前分支。
  • 合并冲突: 出现冲突需手动解决，然后 git add -> git commit。
  • git merge --no-ff: 强制创建合并提交，即使可以快进合并。
  • git merge --abort: 取消当前合并。
  • git merge --squash: 将一个分支的所有提交“压扁”成一个再合并。

8. git rebase - 变基

  • git rebase <base_branch>: 将当前分支的提交“移动”到指定基点之后。
  • git rebase --continue: 解决冲突后继续变基。
  • git rebase --abort: 取消变基。
  • git rebase -i <commit_hash>: 交互式变基，可用于修改、合并、删除提交。
    • 注意: 切勿对已推送的公共分支进行变基操作，会重写历史，导致协作问题。

9. git clone - 克隆远程仓库

  • git clone <repository_url>: 克隆整个远程仓库到本地。
  • git clone <url> <directory_name>: 克隆到指定目录。
  • git clone -b <branch_name> <url>: 克隆指定分支。
  • git clone --depth <number>: 浅克隆，只克隆最新指定数量的提交。
